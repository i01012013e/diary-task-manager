 # Название проекта: Diary Task Manager (Evolution project)

    Fullstack - Приложение. 
        ✔️  Приложение для управления ежедневными задачами и заметками.
        ✔️  Проект создан для демонстрации профессионального роста и навыков 
                архитектуры кода.

------
        
## 🚀 Цель проекта

    ✔️  Показать эволюцию разработки от чистого JavaScript до современных фреймворков.
    ✔️  Проект разделён на 3 этапа рефакторинга. 

------

## 📆 Текущий статус: Этап 1 (Native Focus)

    Технологический стек:
        ✔️ Frontend: Vanilla js (ES Modules), HTML5, CSS3, (BEM).
        ✔️ Backend: Node js (нативные модули `http`, `fs`), PostgreSQL.
        🧰 Инструменты: Git, `env` - для конфигурации.

        Особенности реализации:
            ✔️ Отказ от внешних библиотек (Express и др.) для понимания протокола HTTP.
            ✔️ Работа с БД через нативный драйвер `pg`.
            ✔️ Модульная архитектура на frontend без сборщиков.

------

## Файловая архитектура (Этап 1)

    ```text
        /frontend               - Корневая директория ( frontend ) части приложения 
            /src                - Директория разработки клиентской части приложения  
                /assets         - Директория ( Хранилище ) для стилизации
                    /img        - Директория с картинками
                    main.css    - Главный файл ( CSS ) стилей
                /modules        - Логика ( API, UI, Utils )
                main.js         - Точка входа ( ES Modules )
            index.html          - Главный файл ( HTML )App 
        /backend                - Корневая директория ( Backend ) части приложения 
            /src                - Директория разработки серверной части приложения 
                /config         - Конфигурация ( DB ) 
                /routes         - Роутинг и контроллеры 
                /models         - Запросы - SQL ( Data Access Layer )
                app.js          - Серверная логика для обработки данных 
            server.js           - Стартовый файл для запуска сервера
        /docs                   - Корневая директория по разработке ( SQL )
            schema.sql          - Схема базы данных
    ```
